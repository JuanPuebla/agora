Ordner Struktur:
-----------------

--root
	--modules
		--Downloads
	--pnTemp
		--downloads_cache
		--downloads_screenshots
		--downloads_upload

Der Uploadordner kann jetzt auch ausserhalb des Webroot platziert werden.
Zum Beispiel könnte man einen Ornder Upload erstellen der oberhalb des Webroot liegt und die 3 Downloads Ordner dort hinein verschieben.

In diesem Fall wären folgende Einstellungen für die Pfade zu setzen:

../upload/downloads_upload/
../upload/downloads_screenshots/
../upload/downloads_cache/

Installation:
--------
 - Den Ordner von Downloads 1.31 vom Server löschen
 - Das Downloads 2.x Paket in das Modulverzeichnis der PostNuke Installation kopieren
 - In der Administration von PostNuke unter Module auf Neu erzeugen klicken
 - Modul initialisieren und aktivieren
 - In Administrationsmenü das Modul Konfigurieren


Wichtiger Hinweis:
------------------

Wenn Daten aus dem CmodsDownload oder UpDownload übernommen werden sollen, dann bite aus allen Verzeichnissen dieser Module die .htaccess Dateien löschen die die Module anlegen.  


Upgrade:
--------

 - Das Paket in das Rootverzeichnis der PostNuke Installation kopieren
 - In der Administration von PostNuke unter Module auf Neu erzeugen klicken
 - Den Anweisungen folgen um das Upgrade abzuschliessen


Ordnerrechte:
-------------------
 
 - Folgende Ordner bitt mit Schreibrechten versehen (chmod777)

	--downloads_cache
	--downloads_screenshots
	--downloads_upload


Wenn das Modul entfernt wird werden alle Tabellen in der Datenbank gelöscht!


Upload Ordner ausserhalb des webroot:
-------------------------------------


Um den Upload Ordner ausserhalb des webroot zu platzieren bitte folgendermaßen eintragen ../../pnTemp/downloads_upload/
Jedes ../ hebt den Upload Ordner um eine Level nach oben in der Ordnerstruktur
